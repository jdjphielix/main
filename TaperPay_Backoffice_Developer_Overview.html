🏗 What Is This System

Internal CRM + Onboarding Platform

The TaperPay Backoffice is a full-stack web application that supports the entire Taper commercial pipeline — from first lead contact through to active client management. It is designed to run on a private server and be maintained directly by Claude.

The system manages leads, prospects, onboarding (KYC/AML), compliance email tickets, internal chat, document storage, callback scheduling, and sales analytics — all in one place.

⚠️ This is an internal-only system. It should not be publicly exposed. Run behind a VPN or IP allowlist in production.

🔑 Google OAuth Tokens — What They Are & Why We Need Them

This is the most important integration to understand. We use Google OAuth 2.0 to let users connect their Google account once, after which the system can access Gmail and Google Calendar on their behalf — without ever storing their password.

ACCESS TOKEN

Short-lived credential (1 hour). Used in the Authorization: Bearer … header for every Gmail / Calendar API call. Automatically refreshed when expired — the user never notices.

REFRESH TOKEN

Long-lived credential. Stored encrypted in the users.google_refresh_token column. Used once per access token expiry to get a new access token from Google without requiring the user to log in again.

WHAT SCOPES DO WE REQUEST?
gmail.readonly gmail.send gmail.modify calendar.events userinfo.email userinfo.profile

These scopes allow the system to: read incoming emails for a lead, send compliance replies, create/delete calendar events for callbacks, and identify the authenticated user. No other Google data is accessed.

⚠️ Compliance user note: The compliance@taperpay.com account must complete this OAuth flow once in the system before automated email detection activates. The background Gmail sync uses all active users' tokens — meaning more connected accounts = wider email coverage.

🔄 Background Jobs

Gmail Auto-Sync
Runs every 30 minutes. Syncs Gmail for every active lead (Lead → Prospect → Client stages) using every connected user's token. New emails are automatically linked to compliance cases if a matching thread ID exists.
asyncio task 20 min timeout token pool strategy
System Channel Seeding
Runs on every server startup. Creates and syncs role-based chat channels. Adds users with matching roles, removes users whose roles changed. Fully idempotent.
startup hook idempotent
DB Migrations
Schema changes are applied as ALTER TABLE … IF NOT EXISTS statements on every startup. No separate migration tool — failures are caught, logged, and skipped individually.
startup hook safe on conflict

🌍 Environment Variables

All configuration is loaded from a .env file in the backend root. No secrets are hardcoded.

DATABASE_URL PostgreSQL connection string. Format: postgresql://user:pass@host:5432/dbname REQUIRED
SECRET_KEY JWT signing key. Generate with openssl rand -hex 32. Rotate to invalidate all active sessions. REQUIRED
GOOGLE_CLIENT_ID OAuth 2.0 Client ID from Google Cloud Console. Used for all Gmail + Calendar API calls. REQUIRED
GOOGLE_CLIENT_SECRET OAuth 2.0 Client Secret. Paired with GOOGLE_CLIENT_ID. Store securely — never commit to git. REQUIRED
GOOGLE_REDIRECT_URI Must match exactly what is registered in Google Cloud Console. E.g. https://app.taperpay.com/api/v1/auth/google/callback REQUIRED
ANTHROPIC_API_KEY API key for Claude AI features (call transcript summaries, document scanning, AI assistant). REQUIRED
FRONTEND_URL Frontend origin for CORS. E.g. https://app.taperpay.com — used in CORS middleware. REQUIRED
UPLOAD_DIR Filesystem path for document uploads. Default: ./uploads. Use a persistent volume in production. OPTIONAL

🚀 Running the Platform

Backend (FastAPI)
# Install dependencies
pip install -r requirements.txt
# Copy and fill in .env
cp .env.example .env
# Start server (auto-migrates on boot)
uvicorn app.main:app --host 0.0.0.0 --port 8000
Frontend (React + Vite)
# Install dependencies
npm install
# Development server
npm run dev
# Production build
npm run build
In production, serve the dist/ folder via Nginx or Caddy. Proxy /api/* to the FastAPI backend on port 8000.
💡 Google Cloud Console setup required: Create an OAuth 2.0 Web Application credential. Add your domain to Authorised Origins and your callback URL to Authorised Redirect URIs. Enable the Gmail API and Google Calendar API for the project. The OAuth consent screen must be set to Internal (Google Workspace) or submitted for verification if using external accounts.
